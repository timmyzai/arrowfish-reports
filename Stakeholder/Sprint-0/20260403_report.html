Key Findings (March 2026 Data)

1. Free User Bandwidth Abuse — 13.9 TB/month at $0 Revenue
In March 2026, 11,537 free users consumed 13,854 GB (13.9 TB) of bandwidth — paying nothing. Two free users alone consumed 2.2 TB (1,217 GB + 1,012 GB). 10 free users each exceeded 100 GB. 26 exceeded 50 GB. Free users account for 39% of all bandwidth but generate zero revenue. No server-side enforcement existed — the daily time limit was client-side only and easily bypassed. Of recently active Android users, only 17.7% use the latest version. 928 users on old versions (v1.6.x) were active in the last 2 weeks with zero enforcement. Force-update was never enabled.
2. Revenue Declining — Down 34% in 5 Months
Monthly revenue dropped from ¥51,073 (Oct 2025) to ¥33,718 (Mar 2026) — a 34% decline. Paying users dropped from 2,366 to 1,259 (-47%). App instability, poor trial-to-paid conversion, and 13 crash bugs are the likely drivers. Only 17.7% of recently active Android users are on the latest version. Force-update was never enabled.
3. Platform Stability — 13 Critical Crash Patterns
13 crash/freeze patterns identified: app crash on launch, stuck-connecting, false offline detection, ANR freezes. 7 concurrency bugs found. Users hitting these bugs churn immediately — directly reduces retention and re-purchase rate. Initial fixes being implemented.
4. Infrastructure Waste — ~$735/month Identified
6 "test" nodes still running (~$412/mo). Two redundant proxy servers (~$323/mo) — HK server frequently disconnects. Read-replica database purchased but never connected (only 1 connection since restart). Consolidating to single backend.
5. Security Gaps — Sensitive Data Exposed, Version Control Disabled
Sensitive VPN configuration data found in application logs. Force-update was never enabled — the feature existed but was deliberately disabled. 928 users on old versions (v1.6.x) were active in the last 2 weeks, completely bypassing all usage restrictions.
6. Samsung IAP Non-Functional Since Launch
Samsung app store payments were hardcoded to test mode — zero orders ever processed. Fix prepared. Note: 94% of revenue is WeChat/Alipay; Samsung IAP is a minor channel.

China Connectivity

Taiwan team flagged latency and GFW concerns. Documented but not yet quantified. Data-driven monitoring planned after deployment. Analysis scheduled for May.

关键发现（2026年3月数据）

1. 免费用户流量滥用 — 每月13.9 TB，收入为零
2026年3月，11,537名免费用户消耗了13,854 GB（13.9 TB）的带宽 — 没有支付任何费用。仅两名免费用户就消耗了2.2 TB（1,217 GB + 1,012 GB）。10名免费用户各超过100 GB，26名超过50 GB。免费用户占总带宽的39%，但收入贡献为零。此前没有服务器端限制 — 每日时间限制仅在客户端，容易被绕过。近期活跃Android用户中仅17.7%使用最新版本。928名v1.6.x旧版本用户近两周仍在活跃，完全没有任何限制。强制更新功能从未启用。
2. 收入持续下降 — 5个月下降34%
月收入从¥51,073（2025年10月）降至¥33,718（2026年3月），降幅34%。付费用户从2,366降至1,259（-47%）。应用不稳定、试用转付费体验差、13个崩溃问题是可能的原因。近期活跃Android用户中仅17.7%使用最新版本。强制更新功能从未启用。
3. 平台稳定性 — 13个关键崩溃模式
发现13个崩溃/冻结模式：启动崩溃、连接卡住、误判离线、ANR冻结。发现7个并发问题。遇到这些问题的用户会立即流失 — 直接降低留存率和复费率。初步修复正在实施中。
4. 基础设施浪费 — 每月~$735可节省
6个"测试"节点仍在运行（~$412/月）。两个冗余代理服务器（~$323/月）— 香港服务器时不时无法连线。已购买的读取副本数据库从未使用（重启以来仅1次连接）。正在合并至单一后端。
5. 安全漏洞 — 敏感数据泄露、版本控制被禁用
应用日志中发现敏感VPN配置数据。强制更新功能从未启用 — 928名v1.6.x旧版本用户近两周仍在活跃，完全绕过所有限制。
6. 三星IAP自上线以来无法使用
三星应用商店支付被硬编码为测试模式 — 从未处理过任何订单。修复已准备好。注：94%收入来自微信/支付宝，三星IAP为次要渠道。
