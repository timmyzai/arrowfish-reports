本期成果（按影响排序）

【营收·已修复】专属IP付款 — 全流程修复完成
上周报告的零收入品类，本周完成测试环境全流程验证。同步修复关联缺陷：续费后到期日不刷新（用户需重登才能看到新到期日）。两项均待 QA 双方签核后随正式版上线，专属IP 营收通道恢复。
【营收·已修复】余额支付卡顿 — 流程升级
遗留代码中余额支付走外部网页跳转，用户偶发卡在「处理中」需手动返回。本周由后端新增专用移动端 JSON 接口，App 内直接调用，1 秒内显示结果。当前月余额支付约占总交易 15%，预期减少相关客服咨询与退款申请。
兼容性已验证：旧版本 PC（v2.2.71）与 iOS（v2.2.71）客户端无需更新。
【稳定性·已修复】免费用户启动 / 连接 / 计时器三类深层缺陷
本周收尾上周遗留的稳定性问题：启动卡死（后端字段类型不一致根因修复）、试用计时器漂移与方向错误、切账号后会话残余。用户在正式版将明显感受到：开 App 不卡死、试用时间真实可信、切账号立即可用。
【体验·已修复】错误提示「Api Exception」改为本地化清晰说明
遗留代码所有后端错误统一显示英文「Api Exception / server error」，用户无法理解。本周完成统一错误码映射：设备超过限制、账号被锁、余额不足等场景均有明确中文提示，直接降低客服咨询量。
【收尾·已修复】9 项缺陷 + 9 项体验优化
本周新增修复包含：升级 App 闪退、连接按钮快速点击异常、推荐分享 QR 图片首次安装不显示、空白名单切换静默屏蔽所有线路、收件箱已读状态延迟更新、抽奖弹窗图层重叠等。每项均为独立提交（共 16 个），可单独回滚不影响其他，QA 阶段如发现新问题风险可控。

本阶段后端改动的特殊挑战

后端同时服务 3 类生产客户端，其中 2 类目前无法升级，每次后端改动都必须额外保证向后兼容。
客户端当前生产版本能否升级对后端改动的影响
Android（Arrowfish） v1.7.38 可升级（本周发 v2.0.0） 新接口可正常使用
iOS（HiYou） v1.3.0（已冻结） 不能升级 — Apple 开发者账号被封禁，无法提交 App Store 更新 所有后端接口必须对 iOS v1.3.0 完全兼容；新功能不能假设 iOS 会更新
Windows PC（skytree_pc） v2.2.71（已冻结） 暂时不能升级 — Windows 部署机制尚未研究清楚；原签名方案 WoSign（硬件签名，需配合 Windows SSL Wizard 工具）证书已过期，需研究虚拟签名替代方案后才能再发版 所有后端接口必须对 PC v2.2.71 完全兼容；分页等新功能只能 opt-in 增加，不能改变默认响应格式
本阶段付出的额外成本与下阶段需推进的事项
本阶段：每一项后端改动（余额支付 JSON 接口、交易记录分页、错误码扩展）均额外做了向后兼容设计 + 实测验证。新接口采用「新路径」或「opt-in 查询参数」，原接口完全不动，确保 iOS / PC 用户零感知。开发团队为此付出额外约 30% 的设计与测试时间。

下阶段需推进：(1) iOS Apple 账号申诉 / 新账号注册；(2) Windows 虚拟签名方案研究 + 部署流程文档化。两项不解决，后端长期被「兼容包袱」拖累，重大架构改动会越来越困难。
