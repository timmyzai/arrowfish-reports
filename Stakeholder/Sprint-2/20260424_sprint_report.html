三大类成果

① 营收类
  • 专属IP 付款全流程修复 — 此前因台湾遗留代码缺陷，每笔购买均失败，品类零收入。本阶段修复并验证完成，下周上线后即可入账。
  • 余额支付卡顿修复 — 新移动端 JSON 接口替代外部跳转，1 秒内返回结果。月余额支付约占总交易 15%，预期减少客服咨询与退款申请。
  • 订阅续费日期不刷新 — 关联缺陷一并修复。
② 反滥用 / 反欺诈类
  • 5 层防护机制上线 — 设备维度共享试用额度、卸载重装不发新额度、模拟器检测、IP 注册速率限制（每日 5 个）、单设备账号上限（3 个）。
  • 设备指纹识别替代旧 ANDROID_ID 方案，加密验证，无法伪造。
  • 正常免费用户不受任何影响，每日 30 分钟权益完整。
③ 稳定性 + 体验类
  • 启动卡死修复 — 新装用户因后端字段类型不一致导致启动卡死，已根因修复。
  • 试用计时器三类问题 — 倒计时漂移、方向错误、登出归零异常，统一改为服务器权威。
  • 错误提示本地化 — 「Api Exception」改为明确中文（设备超过限制 / 账号被锁 / 余额不足等）。
  • App 闪退修复 — 升级时点击立即更新概率性闪退已修复。
  • UI 收尾 — 抽奖弹窗图层重叠、空白名单静默屏蔽、收件箱已读延迟、推荐分享 QR 不显示等 9 项体验问题。

本阶段后端改动的特殊挑战

后端同时服务 3 类生产客户端，其中 2 类目前无法升级，每次后端改动都需额外保证向后兼容。
客户端当前生产版本能否升级对后端改动的影响
Android（Arrowfish） v1.7.38 ✅ 可升级（本周发 v2.0.0） 新接口可正常使用
iOS（HiYou） v1.3.0（已冻结） ❌ 不能升级 — Apple 开发者账号被封禁，无法提交 App Store 更新 所有后端接口必须对 iOS v1.3.0 保持完全兼容；新接口不能假设 iOS 会更新
Windows PC（skytree_pc） v2.2.71（已冻结） ❌ 暂时不能升级 — Windows 客户端部署机制尚未研究清楚；原签名方案 WoSign（硬件签名，需配合 Windows SSL Wizard 工具）证书已过期，需研究虚拟签名（virtual sign）替代方案后才能再发版 所有后端接口必须对 PC v2.2.71 保持完全兼容；分页等新功能只能通过 opt-in 方式增加，不能改变默认响应格式
结果：本阶段每一项后端改动（余额支付 JSON 接口、交易记录分页、错误码扩展）都额外做了向后兼容设计 + 实测验证。具体方法：新接口采用「新路径」或「opt-in 查询参数」，原接口完全不动。开发团队为此付出额外约 30% 的设计与测试时间，但确保 iOS / PC 用户零感知。
下阶段需推进：(1) iOS Apple 账号申诉 / 新账号注册；(2) Windows 虚拟签名方案研究 + 部署流程文档化。两项不解决，后端长期被「兼容包袱」拖累，重大架构改动会越来越困难。

八、为什么第3阶段必须先清理技术债 — FFF 复用同一套后端

关键背景：FFF 新产品将复用 orebackend 后端。技术债不修，会被 FFF 继承一遍，问题翻倍。第3阶段的后端通用化不是「锦上添花」，而是 FFF 顺利上线的前提。

接手时发现的固有问题（4–5 天交接窗口的代价）

问题 为什么影响 FFF 第2阶段处理 第3阶段计划
列表接口普遍无分页FFF 高活用户同样会卡顿已修交易记录接口扩展到站内信、邀请、优惠券等
